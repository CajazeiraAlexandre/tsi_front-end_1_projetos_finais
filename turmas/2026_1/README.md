## 📋 Especificações Técnicas dos Projetos Finais

O projeto em equipe consiste no desenvolvimento integral do front-end de um sistema web funcional, baseado nos temas definidos em sala.

### 🛠️ Requisitos Obrigatórios:
* **Estrutura e Semântica:** O projeto deve conter, no mínimo, 5 páginas HTML interligadas de forma lógica. Todas as páginas devem possuir um cabeçalho (`<header>`) com menu de navegação e um rodapé (`<footer>`) padronizados. É obrigatório o uso correto das tags semânticas do HTML5 (`<main>`, `<section>`, `<article>`, `<nav>`, `<aside>`, etc).
* **Estilização Externa:** Todo o código CSS deve estar em arquivos externos (uso de estilo *inline* apenas em casos específicos e devidamente justificados).
* **Layout Responsivo (Grid e Flexbox):** É obrigatório o uso de **Grid Layout** em pelo menos duas páginas para estruturar o layout principal ou organizar galerias/catálogos de itens. É obrigatório o uso de **Flexbox** para alinhamentos unidimensionais (como itens do menu de navegação, alinhamento de ícones e textos dentro de botões). A interface deve se adaptar de forma fluida a diferentes resoluções de tela.
* **Formulários:** Pelo menos uma página deve conter um formulário complexo (como cadastro de usuário ou registro de produto/evento), aplicando as validações nativas do HTML5 e utilizando tipos diversificados de *inputs*.
* **Versionamento e Publicação:** O código deve ser hospedado no GitHub. O histórico de commits deve comprovar a participação de todos os integrantes da equipe. O projeto final deve estar publicado e acessível online via **GitHub Pages**.
